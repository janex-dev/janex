# Janex File Format

The Janex format is a modern executable packaging format for Java programs.

Each Janex file contains the following:

1. A set of classpath and module-path entries. Entries are divided into local entries and remote entries. Local entries are a group of class files and resource files packaged within the Janex file, while remote entries are Maven GAV coordinates that are resolved and downloaded by the Janex tool at runtime.
2. A set of boot metadata, which includes information such as the required Java version and JVM arguments for the program. The Janex tool selects an appropriate Java runtime environment based on this information and starts the program with the necessary JVM arguments.

When launching a Janex format file, the **Janex Launcher** reads the boot metadata to find a suitable Java runtime based on the metadata,
acquires appropriate JVM arguments according to the conditions, and starts a Java process using these JVM arguments.

In the Java process, Janex needs to load a **Janex Boot** provided as a JAR, which provides a custom ClassLoader for `Janex` to load classes from the Janex file.

## Data Types

### Basic Data Types

Janex uses little-endian encoding for all integer and floating-point numbers.

This document uses `u8`/`u16`/`u32`/`u64` to represent 8/16/32/64-bit unsigned integers,
uses `i8`/`i16`/`i32`/`i64` to represent 8/16/32/64-bit signed integers,
and uses `f32`/`f64` to represent 32/64-bit floating-point numbers.

`byte` represents 8-bit opaque data.

### Complex Data Types

This document uses pseudo-code similar to C++ structs to represent complex data types. For example:

```
MyStruct {
    u32         length;
    u8[length]  data;
};
```

Here, `length` is a 32-bit unsigned integer, and `data` is a byte array of length `length`.

`List<T>` represents a variable-length list, where `T` is the type of elements in the list.
The `T` type must define a special value to indicate the end of the list, and this value cannot appear in the list.

## Janex File Structure

The Janex file is the format for executable files generated by the Janex tool. Its overall structure is as follows:

```
JanexFile {
    u64                 magic_number; // 0x0000_0058_454e_414a ("JANEX\0\0\0")
    byte[...]           data_pool;
    BootMetadata        boot_metadata;
    LauncherMetadata    launcher_metadata;
    FileEnd             file_end;
};
```

Where:

- `magic_number`: Used to identify the start of the file and for verification, with a value of `0x0000_0058_454e_414a` (i.e., the string `"JANEX\0\0\0"`);
- `data_pool`: Data pool used to store all information such as class files and resource files, its structure is shown below;
- `boot_metadata`: Boot metadata used to describe the classes and modules that need to be loaded at startup, its structure is shown below;
- `launcher_metadata`: Launcher metadata used to describe the configuration information of the launcher, its structure is shown below;
- `file_end`: File end metadata used to describe the size, offset, etc. of the file, its structure is shown below.


### `FileEnd` Structure

The Janex file is designed to allow arbitrary custom data to be attached at the head of the file (e.g., executable files like PE/ELF, shell scripts, etc.),
so the core metadata of the Janex file is located in the `file_end` field at the end of the file, its structure is as follows:

```
FileEnd {
     u8     magic_number; // 0x0000_0058_454e_414a ("JANEX\0\0\0")
     u4     major_version;
     u4     minor_version;
     u8     flags;
     u8     file_size;
     u8     boot_metadata_offset;
     u8     launcher_metadata_offset;
     byte[32] reserved;
}
```

The `FileEnd` structure is a total of 80 bytes.

Where:

- `magic_number`: Used to identify the start of the metadata, with a value of `0x0000_0058_454e_414a` (i.e., the string `"JANEX\0\0\0"`);
- `major_version`: The major version number of the Janex file format;
- `minor_version`: The minor version number of the Janex file format;
- `flags`: File flags, currently unused, reserved as `0`;
- `file_size`: The total size of the Janex file (including header and tail), the Janex tool uses it to locate the file header;
- `boot_metadata_offset`: The offset of the `boot_metadata` field in the file;
- `launcher_metadata_offset`: The offset of the `launcher_metadata` field in the file;
- `reserved`: Reserved field, currently unused, all bytes are `0`.

### `BootMetadata`

Boot metadata is read by the Janex Boot.

Its structure is as follows:

```
BootMetadata {
    u4 magic_number; // 0x544f4f42 ("BOOT")
    u4 group_count;
    ByteArrayPool stringsPool;
    ResourceGroup[group_count] groups;
}
```

Where:

- `magic_number`: Used to identify the start of the metadata, with a value of `0x544f4f42` (i.e., the string `"BOOT"`);
- `group_count`: Number of groups;
- `stringsPool`: String pool used to store string data, its structure is shown below;
- `groups`: An array of groups, each describing a module or class.
